# このカリキュラムでの扱い

このカリキュラムでは、TodoアプリとSNSアプリの実装解説はReactを基準にします。

Vue.jsでもTodoアプリやSNSアプリは作れます。ただしReact、Vue.js、Next.js、Nuxtのすべてで同じアプリを作ると、学習範囲が広がりすぎます。まずReactで画面開発の共通パターンを身につける方が、後から他のフレームワークに移りやすくなります。

## Vue.jsで理解しておくこと

Vue.js入門では、次の理解を目標にします。

| 学ぶこと | 到達目標 |
| --- | --- |
| `.vue` ファイル | template、script、styleの役割が分かる |
| `ref` | 画面と連動する状態を作れる |
| `v-if` / `v-for` | 条件表示と一覧表示が読める |
| `v-model` | フォーム入力の扱いが分かる |
| props | 親から子へ値を渡す流れが分かる |

## React基準の学びをどう応用するか

Reactで学ぶ次の考え方は、Vue.jsにも応用できます。

- 状態を変えると画面が変わる
- 一覧は配列から作る
- 入力フォームは状態と結びつける
- API通信にはローディングとエラーが必要
- 大きな画面はコンポーネントに分ける

書き方は違っても、作るものの考え方は近いです。

## Todoアプリとの関係

React基準のTodoアプリでは、フォーム、一覧、完了切り替え、API通信を学びます。

Vue.jsで同じことをするなら、次のように置き換えて考えます。

| React | Vue.js |
| --- | --- |
| `useState` | `ref` |
| `onChange` | `v-model` |
| `map` | `v-for` |
| 条件レンダリング | `v-if` |
| props | `defineProps` |

## SNSアプリとの関係

SNSアプリでは、投稿一覧、投稿フォーム、いいね、プロフィール編集、画像アップロードなどをReactで作ります。

Vue.jsで読む場合も、注目する点は同じです。どのデータが状態なのか、どのコンポーネントが責務を持つのか、API通信の失敗をどう表示するのかを見ます。

## よくあるミス

> よくあるミスは、Vue.jsを学ぶためにReactの実践を止めてしまうことです。このカリキュラムの実践ゴールは、まずReactでアプリを作り切ることです。

Vue.jsは、比較して読める状態を作るために学びます。現場でVue.jsのコードを見たときに、Reactの知識と対応づけられれば十分です。
