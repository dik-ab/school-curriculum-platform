# AI開発入門

このセクションでは、**AIを使った開発**を学びます。

このセクションは、TodoアプリやSNSのような大きな実践課題に入る前の準備です。仕様書、既存コード、エラーログをAIに読ませながら、設計・実装・検証を小さく進める方法を学びます。ソフトウェア開発の現場では、**LLM（大規模言語モデル）** を組み込んだAIコーディングツールが急速に普及し、コードを書く・読む・直すという日常の作業のやり方そのものが変わりつつあります。

ただし、AIツールは「魔法の杖」ではありません。仕組みを知らずに使うと、もっともらしい間違ったコードをそのまま受け入れてしまったり、自分で考える力を育てる機会を失ってしまったりします。このセクションのゴールは、AIの仕組みと限界を理解した上で、**AIを「自分の能力を拡張する道具」として使いこなせるようになること**です。

題材として、Anthropic社のターミナル型AIコーディングツール **Claude Code（クロード コード）** を使います。エディタやターミナルから自然言語で指示を出すと、AIがコードの読解・編集・コマンド実行までを行ってくれるツールです。

## 前提知識

このセクションは、ターミナルの基本操作（[ターミナル基本操作](/environment/terminal/)）と、Git（[Git/GitHub基礎](/git/)）を前提とします。AIに変更を任せる以上、「変更前の状態にいつでも戻せる」Gitの習慣が安全装置になるためです。

## 学習の流れ

1. まず「LLMはどういう仕組みで動いているのか」を概念から理解します。仕組みを知ることが、AIの出力を正しく疑い、正しく活用するための土台になります
2. 次にClaude Codeを実際にインストールし、安全に使うための許可モードを学びます
3. CLAUDE.mdとカスタムコマンド・skillsで、AIを「自分のプロジェクトに合わせて調整」する方法を学びます
4. 最後に、計画→実装→レビューという実践的な開発ループと、学習中の皆さんがAIと付き合う上での注意点を整理します

## このセクションのゴール

- LLMの仕組み（トークン、確率的生成）とハルシネーションを自分の言葉で説明できる
- Claude Codeを導入し、許可モードを使い分けながら安全に作業を任せられる
- CLAUDE.mdやカスタムコマンドで、AIをプロジェクトに適応させられる
- AIの出力を必ず自分で検証する、というAI時代のエンジニアの基本姿勢を身につける

ここで学ぶAIとの開発ループは、[Todoアプリ実践](/fullstack-todo/)と[SNS開発](/sns/)でそのまま使います。さらに発展として、[AIチャット開発（RAG）](/ai-chat/)では「AIを使う側」から「AIを組み込んだアプリケーションを作る側」へ進みます。

それでは、[LLMとは何か](/ai/what_is_llm/)から始めましょう。
