# Git/GitHub基礎

ここからは実践開発です。最初のテーマは「バージョン管理」です。

ここまでに、言語の基礎、そしてその言語に対応するフレームワークとデータベース周りの基礎について学びました。これから作るアプリケーションは、ファイル数も変更回数も一気に増えます。「昨日まで動いていたのに、どこを変えたら壊れたのか分からない」「試しに書き換えたいが、元に戻せなくなるのが怖い」「チームメンバーの変更と自分の変更をどう合わせればよいか分からない」——こうした問題を解決するのが、バージョン管理システム **Git（ギット）** と、それをチームで共有するためのサービス **GitHub（ギットハブ）** です。

GitとGitHubは、現代のソフトウェア開発において事実上の標準ツールです。このセクションでしっかり身につけておくと、Todoアプリ実践、SNS開発、CI/CD、デプロイまで、以降のすべての作業でそのまま使い続けることになります。

## 前提知識

このセクションでは、ターミナルでコマンドを入力して作業します。コマンドの基本操作（`cd`、`ls`、`mkdir` など）に不安がある場合は、先に[ターミナル基本操作](/environment/terminal/)を復習してください。また、ファイルとディレクトリの概念（パス、階層構造）は[ファイルとフォルダの基礎](/environment/file_directory/)で扱っています。

## 学習の流れ

1. まず「なぜバージョン管理が必要なのか」「Gitはどういう仕組みなのか」を概念から理解します
2. 次に、自分のPCの中だけで完結するGitの基本操作を、手を動かしながら身につけます
3. ブランチという「並行した作業の流れ」を作り、合流（マージ）させる方法を学びます
4. 最後にGitHubへ作業内容を公開し、実務と同じPull Requestの流れを体験します

概念 → ローカル操作 → ブランチ → リモート共有、という順序です。一つ前のページの内容を使って次のページが進むので、順番どおりに読み進めてください。

## このセクションのゴール

- 自分のプロジェクトをGitで管理し、変更履歴をいつでも振り返れる状態にできる
- ブランチを使って「試行錯誤する場所」と「動く状態を保つ場所」を分けられる
- GitHub上にリポジトリを作り、Pull Requestを通じて変更を取り込む一連の流れを実行できる

これ以降のカリキュラムでは、すべてのコードをGitで管理しながら進めます。特に[Todoアプリ実践](/fullstack-todo/)と[SNS開発（最終プロジェクト）](/sns/)では、機能ごとにコミットし、必要に応じてブランチを切り、GitHub上のリポジトリを起点に開発を進めます。さらに[CI/CD](/cicd/)のセクションでは「GitHubへpushしたら自動でテストが走る」仕組みを作ります。ここで学ぶ内容はその土台になります。

それでは、[Gitとは何か](/git/what_is_git/)から始めましょう。
