# TypeScript基礎

## いよいよTypeScriptに入ります

ここまでHTML/CSS、JavaScriptと学んできたあなたは、すでにWeb開発の基礎を身につけています。そして今、いよいよ**TypeScript**という強力な武器を手に入れる段階に来ました。

TypeScriptは、これからReactでフロントエンドを作るうえでほぼ必須になる言語です。props、state、APIから返るJSON、フォーム入力、コンポーネントの再利用など、React開発の多くの場面で「この値は何の形か」を型で整理します。また、TypeScriptで学ぶ型・関数・オブジェクトの考え方は、Java、Go、Pythonなど他のバックエンド言語を理解するときの土台にもなります。つまりこの章は、フロントエンドを安全に作るための準備であり、他の言語へ進むための橋渡しでもあります。

## ここから少し難易度が上がります

正直に言います。TypeScriptは、これまで学んできた内容よりも少し難しく感じるかもしれません。

「型」という新しい概念が出てきたり、コンパイルという作業が必要になったり、最初は「なんでこんなに面倒なの？」と思うこともあるでしょう。

**でも、安心してください。**

## ここさえ乗り切れば、もう壁は越えたも同然です

TypeScriptを理解できれば：

- **どんなプログラミング言語にもスムーズに移行できます**
  TypeScriptで学ぶ「型」の概念は、Java、C#、Goなど、多くの言語に共通しています

- **Webシステムの前後をつなげて考えやすくなります**
  フロントエンド、Node.jsバックエンド、APIの型定義などを、同じ言語の考え方で整理できます

- **プロフェッショナルな開発者への第一歩です**
  現代の多くの企業がTypeScriptを採用しており、習得すれば即戦力として活躍できます

つまり、**ここは大きな山の一つ**です。この山を越えると、Reactやバックエンド言語の理解がかなり楽になります。

## このセクションで学ぶこと

1. **TypeScriptとは何か** - JavaScriptとの違いを理解する
2. **コンパイルの仕組み** - TypeScriptがどのように動くのか
3. **型の基礎** - 型とは何か、なぜ必要なのか
4. **関数の型** - 関数に型をつける方法
5. **実践的なプロジェクト** - Hello Worldから始める開発
6. **例外処理** - 入力値やAPI通信の失敗に対応する
7. **応用的な型** - ライブラリの型定義を読めるようになる

## 基礎編と応用編の線引き

このセクションでは、ReactやAPI連携に入る前に必要な基礎を優先します。

**基礎編で扱う内容:**

- `string`、`number`、`boolean`、配列、オブジェクト
- 関数の引数と返り値の型
- `type` による簡単な型定義
- `string | null` のような基本的なユニオン型
- `try-catch`、`throw`、`Error`
- `fetch` の `response.ok` 確認

**応用編に回すとよい内容:**

- Generics
- Intersection型
- mapped types、conditional types
- 型ガードの設計
- `as const`、`satisfies`
- `keyof`、`typeof`、indexed access types
- APIレスポンスの実行時バリデーション
- `Result` 型による例外を使わないエラー設計

これらは重要ですが、最初に詰め込みすぎると手が止まりやすい内容です。基礎編では「型を読める・書ける・エラーに対応できる」状態を先に作り、応用編ではReactやNestJS、型定義ライブラリを読む力を伸ばします。

## リンクだけでなく、基礎的な解説も作成しました

このセクションでは、外部のリンクを紹介するだけでなく、**基礎的な内容をかなり丁寧に解説**しています。

一読すると、TypeScriptの全体像がかなり分かるかと思います。焦らず、一つひとつ理解しながら進めていきましょう。

## 学習のポイント

- **完璧を目指さない**
  最初から全てを理解する必要はありません。70%くらい理解できたら次に進んで大丈夫です

- **エラーは当たり前**
  型エラーが出るのは正常です。むしろ、エラーが出ることで問題を早期に発見できます

- **手を動かす**
  読むだけでなく、実際にコードを書いて試すことが何より重要です

- **何度も読み返す**
  一度で理解できなくても大丈夫。分からなくなったら、このページに戻ってきてください
