# このカリキュラムでの扱い

このカリキュラムでは、TodoアプリとSNSアプリの実装解説はReactを基準にします。

NuxtでもTodoアプリやSNSアプリは作れます。ただし、React、Next.js、Vue.js、Nuxtのすべてに対応した実践を作ると、学習量が大きくなりすぎます。まずReactで状態管理、フォーム、一覧、API通信を固める方が、学習の軸がぶれません。

## Nuxtで理解しておくこと

Nuxt入門では、次の理解を目標にします。

| 学ぶこと | 到達目標 |
| --- | --- |
| Vueとの関係 | Nuxtの画面はVue.jsで書くと分かる |
| ファイルベースルーティング | ファイル構成とURLの関係が分かる |
| レンダリング | SSRやSPAという選択肢があると分かる |
| `useFetch` | ページでのデータ取得の雰囲気が分かる |
| server routes | 軽いAPIを置けると分かる |

## React基準の学びをどう応用するか

ReactのTodoアプリで学ぶことは、Nuxtでも考え方として使えます。

```tsx
type Post = {
  id: number;
  body: string;
  liked: boolean;
};

function PostText({ post }: { post: Post }) {
  return <p>{post.body}</p>;
}
```

コードの意味です。

- `Post` は、SNS投稿のデータ型です。
- `id` は、投稿を区別する番号です。
- `body` は、投稿本文です。
- `liked` は、自分がいいねしたかどうかです。
- `PostText` は、投稿本文を表示するReactコンポーネントです。

Nuxtで同じ画面を作る場合も、「投稿データを型で表す」「本文を表示する部品に分ける」「いいね状態をUIに反映する」という考え方は変わりません。

## Todo/SNSをReact中心にする理由

TodoアプリとSNSアプリでは、次の力を優先して伸ばします。

- 画面をコンポーネントに分ける
- フォーム入力を扱う
- APIからデータを取得する
- ローディングとエラーを表示する
- バックエンドAPIとつなぐ

これらはReactでしっかり学べば、Nuxtを読むときにも応用できます。Nuxtでは、そこにVue.jsの書き方、ルーティング、SSR、server routesを追加で理解します。

## よくあるミス

> よくあるミスは、Nuxtを学ぶためにVue.jsとReactの両方の実践を同時に進めようとすることです。最初はReactで1本の実践を完走し、その後にNuxtの構成を比較する方が理解しやすくなります。

もう1つのミスは、フレームワークごとの差分ばかり見て、アプリ設計の共通点を見失うことです。状態、フォーム、一覧、API通信、エラー表示はどのフレームワークでも必要です。
